# docx2md_tui

一个用 Rust 编写的 Windows TUI 小工具，用于把 `.docx` 文档转换为 Markdown，强调可预期的转换流程、写入前确认和边界可见性。

## 0.3 主要能力
- 保持原有 TUI 风格的多阶段流程：输入 -> 转换前摘要 -> 结果预览 -> 写入输出
- 支持单文件转换和目录批量转换（可选递归）
- 支持输出目录配置和明确的批量命名策略
- 支持会话配置记忆：恢复上次输入路径、输出目录、转换选项
- 支持历史记录（`F2` 打开，`Enter` 回填，`Delete` 删除）
- 支持转换进度反馈，避免长任务时界面“卡住感”
- 支持转换结果预览摘要：写入前查看统计、失败项、降级/忽略项和示例内容
- 支持错误分类提示：文件不存在、无效 docx、权限不足、解析失败、I/O 失败等

## 当前转换规则

### 保留
- 普通段落文本
- 标题样式 `Heading1..Heading6` -> `#..######`
- 含编号属性的简单列表段落 -> `- ` 列表项
- 基础表格（无合并单元格） -> Markdown 表格
- 图片资源导出并插入 Markdown 引用

### 降级
- `tab` 和 `br` 以纯文本制表符/换行形式输出
- 含合并单元格的复杂表格降级为纯文本块，并在内容中插入提示

### 忽略
- 脚注/尾注引用（`footnoteReference` / `endnoteReference`）
- 公式对象（`oMath` / `oMathPara`）
- 无法从关系映射解析的图片（会在预览摘要提示为“图片未解析”）

## 图片导出规则
- 图片输出目录：`<当前 md 文件同级>/assets/`
- 命名策略：`image_001.<ext>`、`image_002.<ext>` ...
- Markdown 引用路径：`![image_xxx.ext](assets/image_xxx.ext)`

## 使用说明
1. 启动程序后在输入界面填写：
- 输入路径：支持 `.docx` 文件或目录路径
- 输出目录：可留空（留空时自动推导）
2. 通过快捷键设置关键选项：
- `F3` 覆盖已存在输出
- `F4` 目录递归扫描
- `F5` 批量保留目录结构（关闭后改为平铺命名）
3. 按 `Enter` 进入“转换前摘要”，确认文件数量、输出路径和风险提示
4. 进入“结果预览摘要”，先解析不写盘，查看统计、降级/忽略明细和示例输出
5. 在预览界面确认写入，程序显示进度并输出 `.md` 与 `assets` 资源

## 输出命名策略
- 单文件输入：
- 默认输出目录是源文件所在目录
- 输出名为 `<原文件名>.md`
- 目录批量输入（保留结构开启）：
- 输出为 `<输出目录>/<源相对路径>.md`
- 目录批量输入（保留结构关闭，平铺命名）：
- 输出为 `<输出目录>/<文件名>.md`
- 若平铺后重名，自动追加序号：`<文件名>_2.md`、`<文件名>_3.md` ...

## TUI 快捷键
- 全局：
- `q` / `Esc` 退出（任务执行中时会提示等待完成）
- `Ctrl+V` 粘贴路径
- 输入界面：
- `Tab` 切换“输入路径 / 输出目录”焦点
- `F2` 历史记录
- `F3` 切换覆盖
- `F4` 切换递归
- `F5` 切换保留目录结构
- `Enter` 生成转换前摘要

## 配置与历史存储
- 配置文件：
- `%APPDATA%\docx2md_tui\config.tsv`
- 历史文件：
- `%APPDATA%\docx2md_tui\history.tsv`

## 开发说明
- 入口文件：`src/main.rs`
- 当前项目仍是单文件实现，优先保证功能迭代速度；后续可按 `ui` / `convert` / `storage` 进行轻拆分。

## 已知限制
- 暂不处理富文本样式（字体、颜色、字号、段落对齐等）
- 暂不做公式高保真转换
- 复杂嵌套对象（如文本框、SmartArt）未覆盖
