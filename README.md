# docx2md_tui

一个用 Rust 编写的 Windows TUI 小工具，用于把 `.docx` 文档转换为 Markdown，强调可预期的转换流程和转换前确认体验。

## 主要能力
- 保持原有 TUI 风格的多阶段流程：输入 -> 转换前摘要 -> 结果预览 -> 写入输出
- 支持单文件转换和目录批量转换（可选递归）
- 支持输出目录配置和明确的批量命名策略
- 支持会话配置记忆：恢复上次输入路径、输出目录、转换选项
- 支持历史记录（`F2` 打开，`Enter` 回填，`Delete` 删除）
- 支持转换进度反馈，避免长任务时界面“卡住感”
- 支持转换结果预览摘要：写入前查看统计、失败项和示例内容
- 支持错误分类提示：文件不存在、无效 docx、权限不足、解析失败、I/O 失败等

## 当前转换规则
- 保留：
- 普通段落文本
- 标题样式 `Heading1..Heading6` -> `#..######`
- 含编号属性的简单列表段落 -> `- ` 列表项
- 降级：
- `tab` 和 `br` 以纯文本制表符/换行形式输出
- 忽略（会在摘要和预览阶段提示统计）：
- 图片（`drawing` / `pict`）
- 表格（`tbl`）
- 脚注/尾注引用（`footnoteReference` / `endnoteReference`）
- 公式对象（`oMath` / `oMathPara`）

## 使用说明
1. 启动程序后在输入界面填写：
- 输入路径：支持 `.docx` 文件或目录路径
- 输出目录：可留空（留空时自动推导）
2. 通过快捷键设置关键选项：
- `F3` 覆盖已存在输出
- `F4` 目录递归扫描
- `F5` 批量保留目录结构（关闭后改为平铺命名）
3. 按 `Enter` 进入“转换前摘要”，确认文件数量、输出路径、风险提示
4. 再次确认后进入“结果预览摘要”，先解析不写盘，查看统计和失败项
5. 在预览界面确认写入，程序显示进度并输出 `.md` 文件

## 输出命名策略
- 单文件输入：
- 默认输出目录是源文件所在目录
- 输出名为 `<原文件名>.md`
- 目录批量输入（保留结构开启）：
- 输出为 `<输出目录>/<源相对路径>.md`
- 目录批量输入（保留结构关闭，平铺命名）：
- 输出为 `<输出目录>/<文件名>.md`
- 若平铺后重名，自动追加序号：`<文件名>_2.md`、`<文件名>_3.md` ...

## TUI 快捷键
- 全局：
- `q` / `Esc` 退出（任务执行中时会提示等待完成）
- `Ctrl+V` 粘贴路径
- 输入界面：
- `Tab` 切换“输入路径 / 输出目录”焦点
- `F2` 历史记录
- `F3` 切换覆盖
- `F4` 切换递归
- `F5` 切换保留目录结构
- `Enter` 生成转换前摘要

## 配置与历史存储
- 配置文件：
- `%APPDATA%\docx2md_tui\config.tsv`
- 历史文件：
- `%APPDATA%\docx2md_tui\history.tsv`

## 开发说明
- 入口文件：`src/main.rs`
- 当前项目未拆分多模块，后续可按“UI/转换逻辑/存储”分层。

## 备注
- 当前实现优先保障稳定和可预期，不追求覆盖全部 Word 复杂结构。
- 对图片、表格、脚注、公式的完整转换属于后续增强项。
